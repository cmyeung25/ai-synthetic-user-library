Stage 14 Direction

Backend-driven shell boundary

This review surface keeps the Stage 13 shared controller, but now pushes the product shell one step further by treating the backend snapshot as the visible runtime source of truth.

Snapshot contract

Session state, selected job state, and evidence query focus can now hydrate from one backend workspace-shell payload.

Product-first composition

Plan, run, and evidence surfaces still lead before raw response payloads, but the operator path is now closer to a real shell refresh model.

Still prototype-grade

This is still a static HTML prototype and not yet a framework-hosted shell with persistent app session state or push updates.

Runtime requirement

Use `scripts/start_stage12_demo.bat` first. This stage expects the local API and worker to be current because the page now reads the new `workspace-shell` endpoint instead of stitching review state together in the page.

# Workspace UI Moss Stage 5

This prototype advances the selected `Moss Console` direction into a more product-correct flow:

- removes review-only scenario tabs from the default path
- shows one conversationally inferred study state
- demonstrates how a blocked draft should guide the operator to supplement inputs, confirm a fallback, or save for later
- keeps English and Traditional Chinese review in the same interaction model

Open `demo/workspace_ui_moss_stage5/index.html` in a browser.

What it demonstrates:

- `cannot queue` should not end in a dead disabled button
- the system should guide the next step based on the actual blocking reason
- remediation should preserve study intent and evidence boundaries rather than forcing users into manual reconfiguration

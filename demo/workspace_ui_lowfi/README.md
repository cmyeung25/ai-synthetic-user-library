# Workspace UI Low-Fi Prototype

This low-fi prototype is a review artifact for the planned `Workspace UI Readiness` milestone.

It demonstrates the default flow now defined in the roadmap:

- conversational intake
- progressive clarification
- hidden mode inference
- final research-plan confirmation
- queue and evidence review

How to use:

1. Open `demo/workspace_ui_lowfi/index.html` in a browser.
2. Use the theme switcher to compare `Moss Console`, `Slate Lab`, and `Ink Signal` in the same layout.
3. Click the four phase buttons across the top of the main canvas.
4. Review whether the default path keeps setup simple without hiding the evidence boundary.

What this prototype is for:

- review the default interaction flow
- review the information architecture
- review where advanced controls should stay hidden
- review the flow on top of the shared `workspace-theme.css` design-system foundation

What this prototype is not:

- not a final visual design
- not a real connected frontend
- not evidence that Workspace UI is implemented

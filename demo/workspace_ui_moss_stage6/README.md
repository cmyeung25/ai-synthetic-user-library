# Workspace UI Moss Stage 6

This prototype converges the earlier Moss review artifacts into one more product-correct page:

- conversational intake
- inferred draft plan and evidence check
- blocked confirmation state
- immediate remediation without leaving the same draft

Open `demo/workspace_ui_moss_stage6/index.html` in a browser.

What it demonstrates:

- the default flow can stay in one page from intake through blocked-draft remediation
- queue blocking should happen inside the same study draft, not by bouncing the operator into disconnected review screens
- supplement, downgrade, and save-draft paths can exist as real product actions while scenario-switching stays out of the default path

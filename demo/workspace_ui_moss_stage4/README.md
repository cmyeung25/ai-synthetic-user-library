# Workspace UI Moss Stage 4

This prototype advances the selected `Moss Console` direction into the next Workspace UI story:

- research plan confirmation surface
- explicit queueability versus draft-only versus blocked states
- bilingual review of the same workflow in English and Traditional Chinese
- confirmation payload preview before execution

Open `demo/workspace_ui_moss_stage4/index.html` in a browser.

What it demonstrates:

- the product can keep the main path conversational while still forcing an explicit confirmation checkpoint
- the operator can see what is runnable now versus what should remain draft-only or blocked
- confirmation should preserve user intent, evidence boundaries, and backend limits instead of hiding them
- bilingual UI review can happen on the same interaction model rather than branching the product flow itself

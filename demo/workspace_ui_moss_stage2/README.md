# Workspace UI Moss Stage 2

This prototype advances the selected `Moss Console` direction into the next Workspace UI story:

- conversational intake
- artifact drop
- minimal follow-up prompts
- advanced-controls secondary path
- explicit preflight sidecar

Open `demo/workspace_ui_moss_stage2/index.html` in a browser.

What it demonstrates:

- the default workspace entry can stay conversational without becoming vague
- the system can gather enough structure to draft a research plan before exposing deeper configuration
- the shared design system can evolve by adding reusable intake primitives on top of the same base CSS

## Current Workaround

- Pain: high; switching: medium
- 用 WhatsApp group 傾方向同逐個追覆。
- 自己喺 Instagram、商戶網站同平台之間來回比較時間、地點、價錢。
- 翻朋友之前 send 過嘅 link 再比一比。
- 等朋友私下承諾即刻過數後，先敢代為落訂。

## Trust Boundary

- 要證明係真本地商戶。
- 要清楚列出時間、地點、總價、包含內容、取消／改期安排。
- 要用熟悉付款方式。
- 如果圖片同描述太空泛，佢通常唔會再花時間。

## First Value

- 將揀選項、比較同落訂放埋同一個 flow。
- 可以即刻按時間、地點、價錢篩選，減少開幾個 app 來回比。
- 分享出去嘅 link 已有時間、地點、價錢同可揀時段。
- 朋友撳入去就可以直接揀得唔得，唔使先下載或註冊。
- 睇到邊個已睇、邊個未覆；去到某個人數可以直接落訂或收款。

## Key Insights

- 對呢位 persona，最大痛點唔係『冇活動可揀』，而係多人活動入面點樣快啲定到人、定到時間，同避免自己承擔收數追覆責任。
- Go Out La! 嘅吸引力首先來自整合搜尋、比較同落訂，但真正決定值唔值得用，係佢有冇把群組確認、收款同甩底處理一齊收埋。
- 透明度係基本門檻：清楚時間、地點、總價、包咩、取消／改期安排、本地商戶訊號、熟悉付款方式，缺一樣都會令佢停低。
- 分享功能只有喺『唔使下載註冊、可直接表態、可見未覆、到人數可直接落訂或收款』時先會由假興趣變成真進展。
- 推薦係次要加分，不是主價值；而且一定要可解釋、可關掉，否則容易由方便變成反感。

## Next Experiment

做一個最小可行測試，只驗證『群組活動確認＋收款』而唔係先驗證更多活動發現：用一條免註冊分享連結，畀朋友直接揀時段／表態／付款承諾，並測試對比 WhatsApp 手動追覆，係咪真能縮短確認時間同減少主揪追數步驟。
